# Day 1 — как сейчас выглядит разбор

Дата фиксации: 2026-04-28

## Фактический сценарий (текущая реализация)

После `/start` и выбора проблемы в Onboarding бот переводит пользователя в `analysis`,
после чего в `process_skiller_text()` выдаёт короткий разбор и кнопки:

### Short analysis

```
Ты зависаешь перед стартом.
Это не лень.
Это сбой входа в задачу.
```

Кнопки:
- `Подробнее`
- `Давай действие`

### Long analysis (после «Подробнее»)

```
Паттерн: стоп на входе.
Триггер: неопределенность.
Решение: короткий запуск без давления.
```

Кнопка:
- `Давай действие`

## Где это в коде

- short/long day1 analysis: `skiller_runtime.py`
- общие шаблоны short/long: `templates.py`

## Замечание

Сейчас тексты Day 1 partially hardcoded в runtime.
Чтобы строго соответствовать ТЗ, лучше держать эти тексты в `templates.py` (и/или data-файлах),
а в runtime только подставлять переменные.
